Yang Dibutuhkan

2. Install Node.js & Bun

2.1 Install Node.js 22 LTS

# Install Node.js via NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Verifikasi
node --version   # harus v22.x.x
npm --version    # harus 10.x.x

2.2 Install Bun (Runtime)

# Install Bun
curl -fsSL https://bun.sh/install | bash

# Load bun ke PATH (tambahkan ke ~/.bashrc)
echo 'export BUN_INSTALL="$HOME/.bun"' >> ~/.bashrc
echo 'export PATH="$BUN_INSTALL/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Verifikasi
bun --version   # harus 1.1+

2.3 Install PM2 (Process Manager)

# Install PM2 global
sudo npm install -g pm2

# Verifikasi
pm2 --version
ℹ️ Kenapa PM2?

PM2 menjaga aplikasi Next.js tetap running (auto-restart jika crash, start on boot).

3. Clone & Build Project

3.1 Clone Repository

# Buat direktori aplikasi
sudo mkdir -p /var/www/sayone-portal
sudo chown $USER:$USER /var/www/sayone-portal

# Clone repo
cd /var/www/sayone-portal
git clone https://github.com/pujaasma0721/sayone-hris-portal.git .

# Verifikasi
ls -la   # harus ada package.json, src/, next.config.ts

3.2 Install Dependencies

cd /var/www/sayone-portal

# Install dependencies dengan Bun
bun install

# Verifikasi tidak ada error
echo "Dependencies installed successfully"

3.3 Konfigurasi Environment

# Buat file .env.local
cat > /var/www/sayone-portal/.env.local << 'EOF'
# Backend API URL (HRIS Spring Boot)
NEXT_PUBLIC_API_URL=https://hris.sayone.my.id

# Default company code (fallback jika tidak ada cookie)
NEXT_PUBLIC_COMPANY_CODE=SAYONE

# Set false untuk pakai real backend (bukan mock)
NEXT_PUBLIC_API_MOCK=false
EOF

# Verifikasi
cat .env.local
⚠️ Penting

Ganti NEXT_PUBLIC_API_URL dengan URL backend HRIS Anda. Jangan pakai http://localhost di production.

3.4 Build Production

cd /var/www/sayone-portal

# Build aplikasi
bun run build

# Verifikasi folder .next/ terbentuk
ls -la .next/
echo "Build completed!"
✅ Build Sukses

Jika build berhasil, folder .next/ akan berisi file produksi siap deploy.

4. Start Aplikasi dengan PM2

4.1 Buat Ecosystem File

cat > /var/www/sayone-portal/ecosystem.config.cjs << 'EOF'
module.exports = {
  apps: [{
    name: 'sayone-portal',
    script: 'node_modules/.bin/next',
    args: 'start -p 3000',
    cwd: '/var/www/sayone-portal',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    instances: 1,
    autorestart: true,
    max_memory_restart: '500M',
    error_file: '/var/log/sayone-portal/error.log',
    out_file: '/var/log/sayone-portal/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
};
EOF

# Buat folder log
sudo mkdir -p /var/log/sayone-portal
sudo chown $USER:$USER /var/log/sayone-portal

4.2 Start Aplikasi

cd /var/www/sayone-portal

# Start dengan PM2
pm2 start ecosystem.config.cjs

# Cek status
pm2 status

# Lihat log
pm2 logs sayone-portal --lines 20

4.3 Set PM2 Auto-Start on Boot

5. Konfigurasi Nginx (Reverse Proxy)

5.1 Install Nginx

sudo apt install -y nginx

# Verifikasi
nginx -v

5.2 Buat Site Configuration

sudo cat > /etc/nginx/sites-available/sayone-portal << 'EOF'
server {
    listen 80;
    server_name portal.sayone.my.id;  # Ganti dengan domain Anda

    # Redirect ke HTTPS (aktifkan setelah SSL di-setup)
    # return 301 https://$server_name$request_uri;

    # Next.js app
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # Static files caching
    location /_next/static/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_cache_valid 200 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip compression
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
EOF

5.3 Enable Site

# Enable site
sudo ln -s /etc/nginx/sites-available/sayone-portal /etc/nginx/sites-enabled/

# Test konfigurasi
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx
sudo systemctl enable nginx

# Verifikasi
sudo systemctl status nginx
ℹ️ Ganti Domain

Ganti portal.sayone.my.id dengan domain/subdomain Anda yang sebenarnya.

7. Konfigurasi Backend CORS

Backend HRIS harus mengizinkan origin portal untuk akses API. Pastikan backend sudah dikonfigurasi:

7.1 Update CORS_ALLOWED_ORIGINS

Di server backend HRIS, set environment variable:

# Di /etc/environment atau application.properties backend
CORS_ALLOWED_ORIGINS=https://portal.sayone.my.id,https://hris.sayone.my.id

7.2 Atau Update application-prod.properties

# File: SayOne-HRIS/src/main/resources/application-prod.properties
spring.cors.allowed-origins=https://portal.sayone.my.id,https://hris.sayone.my.id
⚠️ Penting

Jika CORS tidak di-set, browser akan block request dari portal ke backend. Error: CORS policy: No 'Access-Control-Allow-Origin' header

8. Verifikasi & Troubleshooting

8.1 Verifikasi Deployment

# 1. Cek PM2 running
pm2 status
# Output: sayone-portal → online ✓

# 2. Cek Nginx
sudo systemctl status nginx
# Output: active (running) ✓

# 3. Test portal lokal
curl -I http://localhost:3000
# Output: HTTP/1.1 200 ✓

# 4. Test portal via domain
curl -I https://portal.sayone.my.id
# Output: HTTP/2 200 ✓

# 5. Test API connection (dari portal ke backend)
curl https://hris.sayone.my.id/api/public/careers/SAYONE/postings
# Output: {"success":true,...} ✓

8.2 Troubleshooting

Masalah: Portal blank / white screen

# Cek PM2 logs
pm2 logs sayone-portal --lines 50

# Restart aplikasi
pm2 restart sayone-portal

Masalah: CORS error di browser

# Pastikan backend CORS_ALLOWED_ORIGINS include portal domain
# Di backend HRIS server:
echo $CORS_ALLOWED_ORIGINS
# Harus: https://portal.sayone.my.id

Masalah: 502 Bad Gateway

# Aplikasi Next.js mungkin crash
pm2 restart sayone-portal

# Cek port 3000
sudo ss -tlnp | grep 3000
# Harus ada process next-server

Masalah: SSL certificate error

# Renew certificate manual
sudo certbot renew

# Restart Nginx
sudo systemctl restart nginx

Masalah: Portal slug tidak resolve

# Test slug resolve endpoint
curl https://hris.sayone.my.id/api/public/portal/YOUR_SLUG
# Harus return: {"success":true,"data":{"companyCode":"SAYONE",...}}

# Jika 404: backend belum dideploy ulang dengan CareerPortalController
# Jika 400: slug salah atau sudah di-regenerate

9.1 Update Portal (Pull Latest Code)

# Stop aplikasi
pm2 stop sayone-portal
